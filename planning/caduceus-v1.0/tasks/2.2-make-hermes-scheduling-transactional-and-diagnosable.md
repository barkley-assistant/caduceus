# Task 2.2: Make Hermes scheduling transactional and diagnosable

## Outcome and required behavior

Make plugin scheduling transactional and health reporting truthful. Cron
capability is checked before wrapper mutation, every failure leaves a precise
prior state, and `doctor` distinguishes daemon defects from host prerequisites.

## Execution boundaries

- Own `__init__.py` setup, cron install/update/remove, uninstall recovery,
  doctor behavior, plugin tests, and public docs.
- Depend on Tasks 2.1 and 1.4.
- Never read or write `cron/jobs.json` directly.
- Never claim to restart the external Hermes gateway.

## Primary tests

- Bounded cron response, crash-boundary reconciliation, exact rollback, and
  bridge/harness/provider sentinel tests against the pinned-host fixture.

## Acceptance checks

- **2.2-AC-01** — Before mutation, issue a bounded, well-formed `cronjob` list,
  locate the stable Caduceus marker, and snapshot wrapper bytes/mode and the
  matching job.
- **2.2-AC-02** — Reject malformed, denied, timed-out, EOF, crashed, duplicate,
  and foreign-name-collision responses without coercing them to an empty list.
- **2.2-AC-03** — After any create/update/remove ambiguity, re-list: accept the
  exact intended state, retain or restore unchanged state, or compensate both
  wrapper and job to the prior state.
- **2.2-AC-04** — When exact rollback is impossible, return nonzero
  `NeedsAttention` with recovery evidence; never report success or hide a
  wrapper/job inconsistency.
- **2.2-AC-05** — Doctor checks binary, bridge, configuration, cron capability,
  registration, and gateway delivery independently.
- **2.2-AC-06** — Doctor exits 0 when healthy, 1 for Caduceus configuration or
  runtime defects, and 2 for host capability or external prerequisites.
- **2.2-AC-07** — Structured results use `host-capability-unavailable`,
  `gateway-inactive`, `config-incomplete`, or `daemon-defect`; no failure is
  reported as success.
- **2.2-AC-08** — Gateway inactivity reports the exact external restart action
  without attempting it from the plugin.
- **2.2-AC-09** — Install, update, remove, and uninstall recovery are idempotent
  across crashes at every wrapper and job boundary and never access
  `cron/jobs.json`.
- **2.2-AC-10** — Doctor checks configured bridge/harness executability and
  required provider secret-name presence without reading values or making a
  provider network call.
- **2.2-AC-11** — Missing harness/provider readiness is exit 2 external
  prerequisite; configuration/runtime defects remain exit 1.
- **2.2-AC-12** — Remove fake or development-only shipped hook paths and
  pending-work notes; generated wrappers use strict shell behavior,
  deterministic paths, safe quoting/modes, and accurate production comments.

## Handoff expectations

Write `handoffs/2.2.md` from the task template. Record every preflight,
rollback, idempotency, exit-code, category, and gateway-prerequisite result with
the exact host-fixture evidence and forbidden-side-effect checks.
