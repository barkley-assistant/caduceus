# Task 2.1: Implement production configuration bootstrap

## Outcome and required behavior

Replace the production `Config::load` stub with the complete v1 configuration
bootstrap. Setup, status, and run must exercise the same load path and report
actionable bridge, harness, and provider readiness diagnostics.

## Execution boundaries

- Own `src/config.rs`, plugin setup/config integration, and their tests.
- Depend on Tasks 1.1 and 1.4.
- Preserve unrelated YAML, ownership, and permissions; never store or expose a
  secret in generated configuration, diagnostics, or evidence.
- Setup changes configuration only when explicitly invoked.

## Primary tests

- Fail-closed precedence, private atomic rewrite, interruption/retry, redaction,
  and production setup/status/run tests.

## Acceptance checks

- **2.1-AC-01** — Treat a set `$CADUCEUS_CONFIG` as authoritative: missing,
  unreadable, or invalid is an error with no fallback and setup does not create
  that file.
- **2.1-AC-02** — Reject empty or relative `$HERMES_HOME` values with an
  actionable error.
- **2.1-AC-03** — Otherwise check `$HERMES_HOME/config.yaml`, then the XDG
  standalone file; validate any existing source and never overwrite it.
- **2.1-AC-04** — Load generated configuration immediately and resolve the
  default worker command to the user-owned
  `$HERMES_HOME/caduceus/worker-bridge.py`.
- **2.1-AC-05** — Diagnose bridge, harness, and provider readiness separately
  through production setup, status, and run paths.
- **2.1-AC-06** — When no source exists, atomically add minimal non-secret
  configuration only to `$HERMES_HOME/config.yaml`, preserving unrelated YAML
  and metadata, or creating it mode `0600`.
- **2.1-AC-07** — Preserve original owner and mode, never widen permissions,
  use and clean a mode-`0600` temporary file, and redact unrelated YAML secret
  values from diagnostics and evidence.
- **2.1-AC-08** — Prove interruption before and after replacement is retryable,
  including set-but-missing `$CADUCEUS_CONFIG` with no fallback.
- **2.1-AC-09** — Remove configuration stubs and development notes; generated
  configuration text, errors, comments, modes, and paths are accurate,
  production-facing, deterministic, and private.

## Handoff expectations

Write `handoffs/2.1.md` from the task template. Record every precedence and
preservation case, atomic installation evidence, production-path commands,
diagnostic output, changed public surfaces, and residual risks.
