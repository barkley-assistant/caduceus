# Task 1.4: Build the pinned Hermes host fixture

## Outcome and required behavior

Build a hermetic fixture around the actual pinned Hermes Agent v0.18.2 host and
real plugin loader. Cron and gateway capabilities are simulated external
boundaries; the fixture never touches an operator's state. It runs an early
installed-path walking skeleton before Phase 02 implementation begins.

## Execution boundaries

- Own the isolated Hermes test home, pinned-host installation, and simulated
  tool-capability boundary.
- Depend on Task 1.1.
- Simulate external cron and gateway capabilities without claiming a real
  gateway lifecycle.
- Keep the fixture reusable by Phase 02 runtime work and Phase 07 verification.

## Primary tests

- Real pinned-host/plugin load and bounded cron/gateway sentinel tests.

## Acceptance checks

- **1.4-AC-01** — Install and load the plugin through the actual pinned Hermes
  Agent v0.18.2 host.
- **1.4-AC-02** — Simulate well-formed, malformed, denied, timed-out, EOF, and
  crashed `cronjob` capability responses without network or user state.
- **1.4-AC-03** — Represent gateway restart as an explicit external
  prerequisite, never a plugin side effect.
- **1.4-AC-04** — Use no real user home, network, credential, repository, cron,
  or gateway state.
- **1.4-AC-05** — Expose one reusable fixture for Phase 02 and Phase 07 tests.
- **1.4-AC-06** — Execute install `--enable`, record the external restart
  prerequisite, setup, capability-present cron install, doctor, status, and a
  manual run in the isolated `$HERMES_HOME`.
- **1.4-AC-07** — Repeat setup and cron install to observe idempotency, then run
  the capability-absent path.
- **1.4-AC-08** — Record every command, exit code, structured category, and
  artifact in a walking-skeleton report.
- **1.4-AC-09** — Map each expected Phase 02 failure to exactly one later task
  and acceptance ID in the gap register.
- **1.4-AC-10** — Block Phase 01 on fixture/framework failure, an unexpected or
  unowned failure, or silent success; known owned implementation gaps may remain
  failing with deterministic evidence.

## Handoff expectations

Write `handoffs/1.4.md` from the task template. Map every acceptance ID to a
procedure, result, and durable artifact. Record the pinned host version, fixture
isolation, simulated capabilities, commands, side-effect checks, and risks.
Attach the walking-skeleton report and exact gap-register links.
