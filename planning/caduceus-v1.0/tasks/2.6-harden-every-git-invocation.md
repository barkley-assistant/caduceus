# Task 2.6: Harden every Git invocation

## Outcome and required behavior

Harden every Git invocation. Deliver the behavior assigned here and in
`CONTRACTS.md` as one
reviewable GitHub issue or SDD work unit.

## Execution boundaries

- Owned surfaces: Git runner, finalization, and worktree validation.
- Dependencies: 1.2.
- Do not implement deferred v1.x work or modify the v0.1 planning tree.
- Preserve unrelated changes and the established public documentation voice.

## Primary tests

- Git path, process, hook, credential, and containment tests.

## Acceptance checks

- **2.6-AC-01** — Route every Git command through the runner.
- **2.6-AC-02** — Use NUL-delimited byte-safe paths.
- **2.6-AC-03** — Resolve paths before containment checks.
- **2.6-AC-04** — Neutralize ambient hooks and configuration.
- **2.6-AC-05** — Transport PAT credentials through an ephemeral broker without
  argv, URL, configuration, disk, log, or output exposure.

## Handoff expectations

Write `handoffs/2.6.md` from the task template. Map every acceptance ID to
a procedure, observed result, and durable artifact. List changed files, public
contracts, state or schema effects, exact commands, side-effect checks, and
risks.
