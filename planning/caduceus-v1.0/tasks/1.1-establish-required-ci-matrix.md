# Task 1.1: Establish required CI matrix

## Outcome and required behavior

Establish required CI matrix. Deliver the behavior assigned here and in
`CONTRACTS.md` as one
reviewable GitHub issue or SDD work unit.

## Execution boundaries

- Owned surfaces: Actions workflows and required-check documentation.
- Dependencies: 0.4.
- Do not implement deferred v1.x work or modify the v0.1 planning tree.
- Preserve unrelated changes and the established public documentation voice.

## Primary tests

- workflow, commit-message, and canonical CI tests.

## Acceptance checks

- **1.1-AC-01** — Run the canonical gate for pull requests and main.
- **1.1-AC-02** — Test Rust 1.97 and stable with the lockfile.
- **1.1-AC-03** — Run planning validation in CI.
- **1.1-AC-04** — Document checks and artifact retention.
- **1.1-AC-05** — Require the non-empty scope form
  `<type>(<scope>): <description>` from
  [Conventional Commits 1.0.0](https://www.conventionalcommits.org/en/v1.0.0/)
  for every PR commit, or the squash title when squash merge is enforced.
  Type and scope are lowercase; the imperative description has no trailing
  period; and the complete subject is at most 80 characters. The exact valid
  example is `feat(lang): add Polish language example`.

## Handoff expectations

Write `handoffs/1.1.md` from the task template. Map every acceptance ID to
a procedure, observed result, and durable artifact. List changed files, public
contracts, state or schema effects, exact commands, side-effect checks, and
risks.
