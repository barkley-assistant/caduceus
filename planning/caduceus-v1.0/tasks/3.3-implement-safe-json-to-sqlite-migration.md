# Task 3.3: Implement safe JSON-to-SQLite migration

## Outcome and required behavior

Implement safe JSON-to-SQLite migration. Deliver the behavior assigned here and
in `CONTRACTS.md` as one
reviewable GitHub issue or SDD work unit.

**Canonical command.** This task defines the final, normative v1.0
subcommand surface and pin once. The required surface is
`caduceus migrate-state --to sqlite` (see `CONTRACTS.md` STATE-002 and the
v0.1 ↔ v1.0 note there). After this task ships, `MIGRATION.md`,
`docs/state-recovery.md`, and the configuration reference must reflect
the implemented flags verbatim — no `todo!()`, no future-task markers,
no "see 3.3" placeholders. If the implementation requires a different
flag set, the deviation must be recorded in `CONTRACT_REVISIONS.md`
before the task handoff lands.

## Execution boundaries

- Owned surfaces: migration command and importer.
- Dependencies: 3.2, 3.7.
- Do not implement deferred v1.x work or modify the v0.1 planning tree.
- Preserve unrelated changes and the established public documentation voice.

## Primary tests

## Acceptance checks

- **3.3-AC-01** — Lock, back up, validate, import, verify, then activate.
- **3.3-AC-02** — Leave old state active and unchanged on failure.
- **3.3-AC-03** — Require explicit mappings for ambiguity.
- **3.3-AC-04** — Retain a validated source backup.
- **3.3-AC-05** — Dry-run reports a deterministic disposition for every record

- **3.3-AC-06** — State and configuration activation roll back together after
  every cutover interruption.
