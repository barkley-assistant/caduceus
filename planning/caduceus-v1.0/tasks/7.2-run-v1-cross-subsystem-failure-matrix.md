reviewable GitHub issue or SDD work unit.

## Primary tests

## Acceptance checks

- **7.2-AC-01** — Prove GitHub 429 handling and the persisted retry decision.
- **7.2-AC-02** — Prove GitHub 5xx and network-loss recovery without duplicate
  remote effects.
- **7.2-AC-03** — Prove OCI engine loss and orphan reconciliation at every
  lifecycle crash point.
- **7.2-AC-04** — Prove SQLite full and I/O failures preserve valid state and
  recoverable checkpoints.
- **7.2-AC-05** — Prove Hermes startup and shutdown failures drain or preserve
  owned work safely.
- **7.2-AC-06** — Prove Git process, path, authentication, and storage failures
  remain bounded and secret-free.
- **7.2-AC-07** — Prove unavailable finalization lookups remain pending and
  conflicting remote markers transition to `NeedsAttention`.
- **7.2-AC-08** — Prove migration, recovery, retention, and configuration
  cutover rollback at their documented crash points.
- **7.2-AC-09** — Prove concurrency, fencing, repository exclusion, and circuit
  transitions with an injected clock.
- **7.2-AC-10** — Prove the OCI Git-less, secret, mount, network, resource, and
  image-policy boundaries.
- **7.2-AC-11** — Prove an unavailable Hermes `cronjob` tool returns host
  capability failure with no wrapper or registration mutation.
- **7.2-AC-12** — Prove an inactive gateway remains an explicit external
  prerequisite and is never silently reported as healthy.
- **7.2-AC-13** — Prove malformed, denied, timed-out, EOF, crashed, duplicate,
  and foreign-collision cron responses never become empty-list success.
- **7.2-AC-14** — Crash at every wrapper/job boundary and prove exact intended
  state, exact rollback, or surfaced `NeedsAttention` with recovery evidence.
- **7.2-AC-15** — Prove authoritative missing configuration, private rewrite
  interruption, harness executability, and provider-name sentinel failures.

Write `handoffs/7.2.md` from the task template. Map every acceptance ID to

risks.
