# Task 6.3: Enforce isolation policy

## Outcome and required behavior

Enforce isolation policy. Deliver the behavior assigned here and in
`CONTRACTS.md` as one
reviewable GitHub issue or SDD work unit.

## Execution boundaries

- Owned surfaces: mount, secret, network, resource, and upgrade policy.
- Dependencies: 6.2.
- Do not implement deferred v1.x work or modify the v0.1 planning tree.
- Preserve unrelated changes and the established public documentation voice.

## Primary tests

- OCI baseline, secret, Git-less, and upgrade-policy tests.

## Acceptance checks

- **6.3-AC-01** — Default new installs to OCI.
- **6.3-AC-02** — Disable network without a profile.
- **6.3-AC-03** — Keep workers Git-less by masking `.git`, mirrors, and object
  stores while the daemon validates edits and owns every Git operation.
- **6.3-AC-04** — Default-deny OCI secrets and grant only configured per-run
  names through private ephemeral files that are always removed.
- **6.3-AC-05** — Require an explicit upgrade choice.
- **6.3-AC-06** — Run non-root with no capabilities, no-new-privileges,
  read-only rootfs, declared writable mounts, no devices or engine socket, and
  digest-pinned images under an explicit pull policy.
