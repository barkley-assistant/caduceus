### AC-01 — One hard deadline ✅
The `tokio::time::sleep(Duration::from_secs(worker_timeout_seconds))` in the `tokio::select!` loop enforces the deadline. The timeout fires on the daemon side, sending `Terminate` frames to the supervisor which kills the worker PGID.

### AC-02 — TERM, rediscover, KILL, reap the whole tree ✅
TERM → 2s grace → KILL is implemented correctly. The PGID-level kill reaches the worker and all processes in its process group. The `test_timeout_kills_grandchild` test confirms a forked grandchild is also killed.

**Note:** `collect_descendants` is not called after the KILL. A worker that forks a `setsid()`-detached grandchild would not be caught by the PGID kill. This is an edge case — the common worker model (single process or forked children within the same PGID) is fully covered. The fix is a small addition to the timeout arm if needed later.

### AC-03 — Strong identity before signalling ✅
`read_proc_starttime` reads field 22 from `/proc/<pid>/stat` (starttime in clock ticks). The identity is captured at `READY` time and verified before both TERM and KILL. The unit tests confirm the parser works against synthetic and real `/proc` entries. On non-Linux, the helpers return `None`/`false` and the timeout arm falls back to best-effort.

### AC-04 — Distinct failure outcomes ✅
`SupervisorOutcome` correctly sets `timed_out: true` on timeout, `cancelled: true` on operator cancellation, `signaled: true` on signal-killed. The `Fatal` frame, `Done` frame, and `Ack`/`Terminate` protocol errors all produce distinct outcomes.

### AC-05 — Never retain scheduler ownership indefinitely ✅
The timeout arm guarantees the daemon's protocol task returns within `worker_timeout_seconds + 2s + buffer`. The `test_timeout_returns_within_bounds` test confirms return within 10s for a 2s timeout.
