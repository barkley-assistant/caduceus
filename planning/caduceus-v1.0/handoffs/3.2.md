## State/schema effects

## Residual risks

- `PRAGMA journal_mode=WAL` is set on every open. WAL mode allows concurrent
  reads but requires the WAL and SHM files to be present. These are cleaned
  up when the last connection closes.
- `unchecked_transaction` is used because we don't use rusqlite's nested
  transaction feature. This is safe as long as we never nest transactions
  (we don't — each write is a single top-level transaction).
- The `open_in` convenience function joins the state dir with `state.db`.
  The config module will need to be updated (Task 3.7) to use this path
  instead of the JSON file paths.
