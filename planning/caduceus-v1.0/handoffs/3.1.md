## Residual risks

- `atomic_write` creates the parent directory if missing. This is by design so
  that callers don't need to pre-create directories, but it means a typo in the
  path will silently create an unexpected directory tree. This is consistent
  with how `std::fs::write` works.
- The temp file name includes a nanosecond timestamp — there is a theoretical
  collision if called twice within the same nanosecond for the same path. The
  daemon lock prevents this in practice.
- `recover_temp_artifacts` matches files with the pattern `.tmp.<16-hex-chars>`.
  A legitimate file whose name ends with this pattern (extremely unlikely) would
  be removed. This is an acceptable risk for the recovery utility.
