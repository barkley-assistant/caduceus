## Risks

- The async drain uses `Arc<Mutex<BoundedTranscriptWriter>>` with `spawn_blocking` per chunk. If a chunk is very large (>4KB) the serialization through the mutex may cause a brief stall. The 4KB chunk size is the same as the existing drain.
- `finalize()` returns `Err` on truncation, which causes the supervisor to return `Err` to the daemon. This is the correct behaviour per AC-03 — a truncated transcript is a failed run. Operators should set `transcript_max_bytes` high enough for their workload (default 10 MiB).
- The `--transcript-max-bytes` flag is new in the supervisor command; older supervisors without it will fail. The `build_supervisor_command` always passes it, so this is only a risk during a rolling upgrade.
