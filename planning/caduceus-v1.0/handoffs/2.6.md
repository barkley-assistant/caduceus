## Changes summary

### `src/worktree.rs` — Runner hardening

1. **`GitOutputRaw` struct** (lines ~63-82): Parallel to `GitOutput` with raw `Vec<u8>` stdout
   for NUL-delimited `-z` output parsing.
2. **`run_in_raw` method** (lines ~308-450): Same async wait loop as `run_in` but returns
   `GitOutputRaw` with uncapped stdout bytes.
3. **`build_command` hardening** (lines ~535-605):
   - `-c core.hooksPath=/dev/null` prepended before user args (AC-04)
   - `GIT_CONFIG_NOSYSTEM=1` env var (AC-04)
   - `GIT_DIR`/`GIT_WORK_TREE` set from `cwd` when present (AC-04)
   - `GIT_ASKPASS`/`GIT_ASKPASS_FD` set when credential fd present (AC-05)
4. **Credential broker** (lines ~165-244, ~475-503):
   - `GitRunnerInner.credential_helper_fd: Option<OwnedFd>` (Unix only)
   - `setup_credential_broker`: creates anonymous pipe, writes PAT, closes write end
   - `write_credential_helper`: writes shell one-liner to tmp, chmod 700
   - Non-Unix: `cfg(not(unix))` returns `None` (daemon refuses non-Unix anyway)

### `src/finalize.rs` — Function conversion

1. **`git_status_v2`**: Uses `runner.run_in_raw` with `--porcelain=v2 -z`, raw bytes
   split on NUL (same parsing logic, just from `GitOutputRaw`).
2. **`git_rev_in`**: Sync wrapper calling `git_rev_in_async` via `drive_block_on`.
   Async version `git_rev_in_async` for use from `push_daemon_branch`.
3. **`git_add`**: Uses `runner.run_in` via `drive_block_on`.
4. **`git_commit`**: Uses `runner.run_in` via `drive_block_on`.
5. **`is_ancestor`**: Uses `runner.run_in` with `.await` (already async).
6. **`drive_block_on`**: Falls back to creating a fresh runtime if no Tokio handle
   is available (needed for sync test contexts).
7. **AC-03 symlink escape**: Adds `std::fs::canonicalize` before containment check,
   rejects paths that resolve outside the worktree root.

## Risks

- **Credential broker is Unix-only**: The `cfg(unix)` gate means the broker is absent
  on non-Unix platforms. The daemon already refuses non-Unix, so this is a
  completeness gap only.
- **`drive_block_on` runtime creation**: When called from a sync context without a
  Tokio runtime, each call creates a new `current_thread` runtime. This is O(1) per
  git call so acceptable for the 4-5 calls in `commit_code_result`.
- **Helper script on disk**: `write_credential_helper` writes a shell script to
  `/tmp/caduceus-askpass-<pid>`. The script contains no secrets (only fd number).
  It is not cleaned up after the process exits (OS temp dir cleanup handles this
  on normal systems).
