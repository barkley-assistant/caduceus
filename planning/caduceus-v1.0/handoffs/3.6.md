## State/schema effects

Old backup and corruption-archive files may be removed from the state directory.
Active state files are never modified.

## Forbidden side-effect checks

- No active state files modified.
- No daemon lock acquired.
- No GitHub API calls.
- No git mutations.

## Residual risks

- The retention window is measured by file modification time, not by any
  internal timestamp. An operator who touches a backup file (`touch`) will
  keep it alive longer than expected.
- The function does not acquire the daemon lock. This is intentional —
  pruning old backup files is safe to run concurrently with a tick.
