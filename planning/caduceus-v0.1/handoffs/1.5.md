## Public signatures/contracts used

- **`CaduceusError`** is the canonical cross-cutting error type. The
  variant set and the field semantics are pinned by `CONTRACTS.md`
  "Error contract".
- **`VoiceError`** is a separate type used by the public-voice
  validator (Task 6.6). Keeping it distinct lets the validator
  report a precise reason (forbidden term or size limit) without
  conflating with daemon-wide errors.
- **`scrub(value) -> String`** is the canonical redaction helper.
  Callers constructing errors must pre-scrub before passing a string
  into a variant field; `Debug` also scrubs every string field as a
  second line of defence.
- **`CaduceusError::other(message)`** is the only public
  constructor for the catch-all `Other` variant.

## Behaviour

- **`Display`** is generated by `thiserror` from per-variant
  `#[error]` format strings. Callers must pre-scrub string fields;
  the contract for `Display` is the same as for any human-readable
  string in the daemon.
- **`Debug`** is a custom impl that scrubs every string field
  through `scrub`. `Debug` never contains a resolved token. This
  is the layer that protects panic messages, `unwrap` failures,
  `Result::unwrap_err`, and `tracing::error!` payloads.

  `Display`, `Debug`, `Eq`, and `is_terminal()`.

  for the `Other`, `Io`, `Git`, and `CADUCEUS_GITHUB_TOKEN`
  variants. `scrub()` is exercised for bare/quoted assignments,
  sub-prefix identifiers (`MY_GITHUB_TOKEN` is preserved), empty
  input, and multiple assignments per line.

## Forbidden-side-effect checks

Per CONTRACTS.md and the Phase 01 gate:

- `CaduceusError` is a plain enum with no `Drop` or external
  resources. Constructing one is allocation-free (modulo the
  owned strings).
- `Debug` scrubs every string field through the deny-list helper.
  The test suite verifies that the four most common leak vectors
  (`Other("GITHUB_TOKEN=...")`, `Io(...)`, `Git { stderr: ... }`,
  `CADUCEUS_GITHUB_TOKEN=...`) all come back scrubbed.
- `scrub` is a pure function; tests cover bare and quoted
  assignments, empty input, sub-prefix identifiers, and multiple
  assignments per line.

## Residual risks

- `Display` does not scrub; it is the caller's responsibility to
  scrub before constructing an error. The structured logger
  uses `Debug` (which always scrubs); `Display` is for end-user
  surfaces where the operator has already curated the message.
- `scrub` only handles the three documented credential names.
  Operators that introduce new credential variables must add
  them to `scrub`'s deny-list in the same commit that adds the
  variable to `Config::DENIED_ENV_VARS`.
