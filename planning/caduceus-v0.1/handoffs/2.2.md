## Public signatures/contracts used

- **`poll::discover_watched_repos(client: &Client, cfg: &Config) -> CaduceusResult<Vec<String>>`**
  is the public entry point. When `cfg.watched_repos` is non-empty, no
  HTTP call is made; the configured slugs are validated, deduplicated
  case-insensitively (the first canonical-cased occurrence wins), and
  sorted by the lowercase key. When `cfg.watched_repos` is empty, the
  daemon paginates `/user/repos?per_page=100&sort=full_name`, follows
  `Link: <…>; rel="next"`, and stops on `rel="last"` or the absence of
  the header.
- **`poll::MAX_PAGES_PER_ENDPOINT = 20`** is the hard cap that the
  contract attaches to every paginated endpoint.
- **`poll::REPOS_PER_PAGE = 100`** is the documented per-page row
  count so a single page always fits the GitHub envelope.
- **`poll::next_url_from_link_header(header: &str) -> Option<String>`**
  is the public Link-header parser. Exposed so the test suite can
  drive it without a network fixture.
- **`github::Response`** now carries `headers: HeaderMap` so the
  pagination loop reads `Link` and the rate-limit observer reads
  `x-ratelimit-remaining`/`x-ratelimit-limit`/`x-ratelimit-reset`.

## Behaviour

- **Configured bypass.** When `cfg.watched_repos` is non-empty, the
  result is the configured list with each entry validated through
  `is_valid_repo_slug`, deduplicated case-insensitively, and sorted
  by lowercase key. The first canonical-cased occurrence of each
  slug is kept; subsequent casings are dropped. No HTTP request is
  issued and the persistent cache is not consulted.
- **API discovery.** When `cfg.watched_repos` is empty, the daemon
  paginates `/user/repos?per_page=100&sort=full_name`. Each page
  must be a JSON array of repository objects; non-array bodies are
  a typed error. Each object must carry a valid `full_name`;
  missing or invalid slugs are a typed error. Repositories with
  `archived: true` or `disabled: true` are silently dropped.
  Discovery is bounded by `MAX_PAGES_PER_ENDPOINT = 20`; exceeding
  the cap is a typed error, never silent truncation.
- **Pagination.** The Link header parser extracts the URL marked
  `rel="next"`. The header is parsed segment by segment, allowing
  for both the documented `rel="next"` and `rel="last"` form, and
  returns `None` when no `rel="next"` is present (signalling the
  last page).
- **Rate-limit awareness.** Any page that returns
  `x-ratelimit-remaining: 0` plus a `x-ratelimit-reset` Unix
  timestamp surfaces a typed `CaduceusError::RateLimited` with
  `reset_at` computed as `max(0, reset - now)` seconds. A
  successful response with `x-ratelimit-remaining > 0` is treated
  as normal. This is the minimum surface Task 2.4 needs to persist
  observations; the meta-layer wiring lives there.
- **Stable output.** After pagination the result is sorted in
  ASCII order so callers can diff snapshots across ticks without
  being affected by GitHub's response ordering.
- **Persistent cache reuse.** All GETs go through the Task 2.1
  client and its `HttpCache`. A repeat tick with an unchanged
  repository set is fully served from the cache.

## Tests added or changed

`tests/repository_poll_test.rs` — 12 tests, all green:

- **Configured bypass** — non-empty `watched_repos` skips the API
  call; case-insensitive dedup keeps the first canonical casing;
  sort is by lowercase key.
- **Configured bypass sort** — `["octocat/zzz", "OctoCat/AAA",
  "mona/BBB"]` returns `["mona/BBB", "OctoCat/AAA",
  "octocat/zzz"]`.
- **Empty result** — `[]` body yields an empty `Vec<String>`.
- **Archived/disabled exclusion** — repos with `archived: true` or
  `disabled: true` are silently dropped; the others are returned.
- **Malformed object** — a page with one object missing
  `full_name` is a typed error naming the missing field.
- **Malformed JSON** — a non-array body is a typed error.
- **Two-page traversal** — page 1 returns two repos plus a
  `rel="next"` link; page 2 returns the third repo; the merged
  result is all three.
- **Rate-limit on page two** — page 1 succeeds, page 2 carries
  `x-ratelimit-remaining: 0`; the call surfaces
  `CaduceusError::RateLimited { reset_at ≤ 600s, remaining: 0,
  limit: Some(5000) }`.
- **Page cap** — a server that always emits `rel="next"` trips
  `MAX_PAGES_PER_ENDPOINT` and surfaces a typed error.
- **Link header parser** — extracts `rel="next"` from a multi-rel
  header; returns `None` when only `rel="last"` is present or the
  header is empty.

## Forbidden-side-effect checks

Per CONTRACTS.md and the Phase 2 task packet:

- The configured bypass performs **no** HTTP request — verified by
  routing the test through a path that would 404 if a request was
  attempted.
- The API path performs at most `MAX_PAGES_PER_ENDPOINT = 20` page
  fetches per discovery call — verified by the page-cap test.
- Archived and disabled repos are dropped silently (no error); the
  `archived/disabled` exclusion test asserts only the four
  eligible slugs survive.
- A missing `full_name` is a typed error that names the offending
  page number — no silent truncation, no best-effort parsing.
- Rate-limit exhaustion is a typed `CaduceusError::RateLimited`,
  not a plain HTTP error, so the meta layer (Task 2.4) can
  persist the observation before the daemon exits.
- All HTTP traffic goes through the Task 2.1 client, so token
  redaction, redirect policy, and the 10 MiB body cap all apply
  unchanged.
- No subprocess execution, no filesystem mutation outside
  `<state_dir>/cache/http.json`, no env mutation. The test suite
  creates tempdirs only.

## Residual risks

- **Rate-limit pre-flight.** Task 2.2 surfaces a typed
  `RateLimited` when a *response* indicates exhaustion. The
  daemon's *preflight* "no later tick performs GitHub calls before
  the persisted reset time" (CONTRACTS.md invariant 11) is owned
  by Task 2.4 and is not implemented here. Until then a freshly
  discovered rate-limit error short-circuits the current call but
  does not yet prevent the next tick from trying.
- **Discovery ordering.** The configured bypass sorts by the
  lowercase key, so `mona/BBB` precedes `OctoCat/AAA`. The
  contract does not pin a specific sort order; the chosen order
  is the natural ASCII order once case is normalised.
- **Empty-200 fallback.** When GitHub returns a 200 with a body
  that is not a JSON array, the discovery loop returns a typed
  "JSON parse" error. The contract does not specify the failure
  shape; we chose to surface the error rather than silently
  yielding an empty list, so operators see the regression
  immediately.
- **Per-page row count.** `REPOS_PER_PAGE = 100` matches the
  documented GitHub maximum; lowering it would not break the
  contract but would inflate the page count and risk tripping
  `MAX_PAGES_PER_ENDPOINT` for operators with many repos.
