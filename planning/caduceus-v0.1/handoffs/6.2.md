## Public surface

| Symbol | Kind | Purpose |
|---|---|---|
| `PushMode` | enum | The four-state outcome of the push: `Created`, `AlreadyCurrent`, `FastForward`, `Diverged`. `Diverged` is reached only via the `PushCollision` error, never via the success return. |
| `PushOutcome` | struct | `mode` + `branch` + `remote_oid`. |
| `push_daemon_branch(ctx, runner)` | async fn | The high-level entry point. Queries the remote, picks the right push mode, returns a `PushOutcome`. |
| `push_and_finalize(ctx, runner)` | async fn | Thin shim that returns `FinalizeOutput { action: Pushed, … }`. |
| `CaduceusError::Push` | variant | `ls-remote` or `git push` failure with redacted stderr. |
| `CaduceusError::PushCollision` | variant | The remote is a non-ancestor; the push is rejected without `--force`. |

## Behaviour (per the contract)

1. **No PAT in arguments, URLs, or environment.** The function only passes the branch ref name and the remote URL; the runner's `GITHUB_TOKEN` / `CADUCEUS_GITHUB_TOKEN` / `GH_TOKEN` env scrubbing handles authentication. The integration test `push_never_places_pat_in_arguments_or_url` asserts the URL is recorded verbatim and contains no token-shaped string.
2. **Query before push.** The function runs `git ls-remote --heads <remote> <branch>` to discover the remote OID (or its absence) before deciding the action.
3. **Four contract cases.**
   * **Absent**: `git push <remote> <local>` creates the ref. `PushMode::Created`.
   * **Identical**: no-op success. `PushMode::AlreadyCurrent`.
   * **Ancestor**: `git push` fast-forwards. `PushMode::FastForward`. The ancestor check uses `git merge-base --is-ancestor <remote> <local>`; a `true` exit means a fast-forward.
   * **Diverged**: rejected with `PushCollision`. **No `--force` is ever used**; the orchestrator surfaces the collision so the operator reconciles the branch manually.
4. **Persist `Pushed` immediately after success.** The `push_daemon_branch` function returns the `PushOutcome` synchronously, so the orchestrator can persist the `FinalizationCheckpoint { stage: Pushed, branch, remote_oid }` as soon as the call returns.
5. **Bounded stderr with secrets redacted.** The `run_push` and `ls_remote_branch` helpers apply `error::scrub` on the captured stderr before wrapping the failure in a `CaduceusError::Push`. The integration test `push_redacts_secret_in_stderr` confirms the scrub helper redacts the canonical `GITHUB_TOKEN=…` shape.
6. **Timeout via the runner.** The function calls `runner.run_in` / `runner.run`, both of which honour `git_timeout_seconds`. A hanging remote or credential helper is killed via the runner's process-group SIGKILL (Task 4.1). The integration test `push_uses_runner_timeout_for_hanging_remote` pins the timeout-config contract; the actual hang-cancellation is exercised by the runner's own tests in `worktree_test.rs`.

## Forbidden side-effects (per CONTRACTS.md invariants)

- **No force-push.** A diverged ref returns `PushCollision`; the function never passes `--force` or `+` to git push.
- **No PAT in arguments, URLs, or env.** The function is URL-only; the runner handles auth.
- **No label change.** Push is git-only.
- **No worktree teardown.** The orchestrator tears the worktree down after the push step lands.
- **No GitHub HTTP call.** The push goes through git; the daemon never constructs a GitHub API request of its own.

## Residual risks

- The `git_timeout_seconds` is applied to **each** subcommand (`ls-remote`, `push`). For a network that hangs slowly, the daemon may still report `Push` errors with a non-zero `timed_out` flag; the orchestrator should treat both `Push` and `PushCollision` as recoverable on a later tick. The integration test pins the timeout-config contract; the actual hang-cancellation is covered by the runner's tests.
- The push runs through the runner's default `git_timeout_seconds` value (300 by default). A tighter timeout can be configured via `RawConfig::git_timeout_seconds`; the test `push_uses_runner_timeout_for_hanging_remote` constructs a 1-second timeout to assert the contract.
- A `file://` URL in tests is a local-only proxy for a real `https://` remote. The runner's `git credential helper` path is the production auth; the test's `file://` URL exercises the git refspec machinery without needing a network. Phase 6 will add a `git credential helper` integration test.
- The `merge-base --is-ancestor` command requires the local repo to have the remote OID fetchable. For a `file://` URL this is automatic; for a real `https://` URL, the orchestrator should fetch before the push. Phase 6's orchestrator is responsible for that fetch.
