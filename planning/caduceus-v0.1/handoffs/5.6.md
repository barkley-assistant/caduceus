## Public signatures/contracts used

- **`WorkerContext { schema_version, issue, issue_title, issue_body, labels, comments, trusted_comments, events, truncation, built_at }`** is the canonical stable context the worker reads via `CADUCEUS_CONTEXT_JSON`. Every field is `serde::Serialize + serde::Deserialize`.
- **`ContextComment { author, body, created_at }`** mirrors the worker-visible shape of an `IssueComment`; `body` is bounded to `MAX_COMMENT_BODY_BYTES = 64 KiB` per the contract, with a `...<truncated N bytes>` marker when truncated.
- **`ContextEvent { kind, actor, created_at, label_name }`** mirrors the worker-visible shape of an `IssueEvent`.
- **`ContextTruncation { comments_truncated, trusted_comments_truncated, events_truncated, dropped_untrusted_comments, dropped_trusted_comments, dropped_events, body_truncated_count, total_body_bytes_dropped }`** carries the metadata so the worker can see *exactly* what was dropped.
- **`BuildInputs<'a> { config, detail }`** is the documented entry point. The struct is `Debug`-able so the controller can log a redacted summary if it needs to.
- **`build_context(BuildInputs) -> CaduceusResult<WorkerContext>`** is the canonical builder. It is pure: it reads only the inputs and returns a serialisable struct. The truncation logic is split into `reduce_events_to_budget` and `reduce_comments_to_budget` so each rule is testable in isolation.
- **`encode_context(&WorkerContext) -> CaduceusResult<String>`** is the canonical serializer. It applies a final size check so the JSON is guaranteed to fit in `MAX_CONTEXT_BYTES = 1 MiB`.
- **`decode_context(&str) -> CaduceusResult<WorkerContext>`** is the canonical deserializer. Used by the round-trip test.

## State/schema effects

- `Regex` compilation already happens at config-load time (Task 1.1); this task consumes the compiled regexes via `Config::compiled_ignore_patterns`.

## Residual risks

- **`reduce_comments_to_budget` is O(n²) in the worst case.** Each
  iteration serializes the full comment list via the real
  `serde_json` encoder to get a byte-accurate measurement; in
  practice n shrinks quickly, but the
  `comments_drops_untrusted_before_trusted_on_size_cap` stress
  test (600 comments) takes ~25 seconds. A future revision can
  switch to a `serde_json::to_writer` / `CountWrite`-style
  streaming counter for a constant-factor speedup. The contract
  is satisfied; performance is the only outstanding issue.
- **The trust partition is recomputed from `all` rather than
  maintained incrementally.** A future revision can avoid the
  second pass by tracking which indices in `all` survive.
- **No `deny_unknown_fields` on the context struct** because
  workers may evolve to consume additional optional fields.
  The schema version field is the contract-pin.
