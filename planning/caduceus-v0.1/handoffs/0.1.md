## Public signatures/contracts used

- `pub use` from `lib.rs`: only the bounded list called out in Task 0.1
  ("`CaduceusError`, `CaduceusResult`, `IssueDetail`, `IssueKey`, queue
  enums/entries, and `WorkerResult`") and the immediate queue-derived
  types (`Phase`, `TicketType`, `FinalizationStage`,
  `FinalizationCheckpoint`, `QueueEntry`, `QueueState`) that downstream
  tasks consume. No `pub use` re-export touches `config`, `context`,
  `github`, `poll`, `validate`, `verify`, `worktree`, `worker_supervisor`,
  `migrate`, `status`, `meta`, `logging`, `prompt`, or `finalize` — those
  remain reachable through their canonical paths, per Task 0.1's "modules
  otherwise use their canonical paths" rule.
- `Config::test_defaults(root: &Path)` exists in lieu of any
  host-dependent `Config::defaults()` (CONTRACTS.md "Configuration":
  "Tests use `Config::test_defaults(root: &Path)`, never a
  host-dependent `Config::defaults()`").
- The CLI router implements the no-argument rewrite via `args_os()` —
  no `default_subcommand` clap annotation. A bare `caduceus` invocation
  silently forwards to `run`.

## Residual risks

- Module bodies are deliberately stubbed (typed return values, no real
  I/O). Later tasks must replace these stubs without re-shaping the
  public types, or the bounded re-export list in `lib.rs` will break.
- `WorkerResult` is the only worker-side type re-exported; later tasks
  may need to add more (e.g. the rendered PR text), in which case
  `lib.rs` should be the single point of edit and the manifest's
  `lib.rs owns` list should be honoured.
- `Config::load()` returns an error for now; nothing in the workspace
  depends on it yet, so the error never reaches the user. Task 1.1
  must replace the body without changing the error type.
- `cli::run` is a no-op after parsing — the real per-subcommand bodies
  are filled in across phases 1–7. The CLI contract is preserved
  (silent `run`, no version banner, exit codes) as bodies land.
- The `tokio = "full"` feature selection is broad. If the eventual
  feature set is narrower, a contract-aware tightening is appropriate
  in a later phase rather than at scaffolding.
