## Public signatures/contracts used

- **`init(log_path) -> CaduceusResult<LogGuard>`** installs
  tracing's process-global subscriber. The first successful call
  sets a sticky `GLOBAL_INITIALISED` flag (per `tracing`'s "one
  global subscriber per process" rule); subsequent calls return
  `CaduceusError::Config("logging already initialised")` and never
  try to install a second subscriber.
- **`LogGuard`** owns the non-blocking writer's `WorkerGuard`.
  Dropping it flushes pending events and shuts the writer down; the
  global flag remains sticky for the life of the process.
- **`init_for_test(log_path, body)`** wraps a JSON subscriber in
  `tracing::subscriber::with_default` so unit tests can drive
  logging in parallel without touching the global default.
- **`build_test_subscriber(writer) -> impl Subscriber + Send + Sync`**
  exposes the JSON layer builder so tests that need to inspect
  emitted events directly don't need to load the daemon's full
  subscriber stack.
- **`redact(value) -> String`** replaces `GITHUB_TOKEN=...`,
  `CADUCEUS_GITHUB_TOKEN=...`, and `GH_TOKEN=...` assignments (bare
  or quoted) with `<redacted>` while keeping the variable name.
  Sub-prefix identifiers (e.g. `MY_GITHUB_TOKEN`) are preserved —
  the deny-list matches variable names, not substrings.

## Behaviour

- File → compact JSON lines (`tracing-subscriber::fmt::layer().json()`).
- Stderr → human-readable warnings/errors. The stderr layer's
  filter is at `LevelFilter::WARN`, so info events stay in the file
  but never echo to the operator's terminal.
- Parent directories are created with mode `0700` to match the
  daemon's state-dir contract.
- `tracing::info!`, `tracing::warn!`, and `tracing::error!` flow
  through both layers. The `caduceus_*!` macros are thin wrappers
  over the tracing macros; they exist so call sites read cleanly.

## State/schema effects

- `init` only reads/writes the log file and its parent
  directories. No persistent state outside the file.
- The global flag and tracing's process-wide subscriber are
  process-local; they do not survive a daemon restart.

## Forbidden-side-effect checks

Per CONTRACTS.md and the Phase 01 gate:

- `init` only mutates the log file and its parent directory.
  Existing parents are preserved; no recursion past the log
  file's parent.
- `init_for_test` is the only path for tests to exercise logging
  in parallel; it never touches the global default.
- `redact` is deterministic and side-effect-free; tests run it on
  every code path (bare/quoted/sub-prefix/multiple).
- The `try_init` failure path is wrapped to mark
  `GLOBAL_INITIALISED` so a follow-up call in the same process
  fails fast without re-entering tracing's `try_init`.
- No network access; no subprocess invocation; no cron work.

## Residual risks

- Tracing's process-global default is genuinely permanent for the
  life of the process. Tests that need a "clean slate" must run
  in a fresh process; the test suite is structured so the
  single init install runs once under `INIT_LOCK` and the other
  serial tests cover the failure surface.
- The redaction list is hard-coded to the three documented
  credential names. Operators who introduce new credential
  variables must add them to `config.rs::DENIED_ENV_VARS`; the
  adapter redaction layer is intentionally separate so the
  logging scope and the env-allowlist scope can evolve
  independently.
