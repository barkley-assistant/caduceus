## Public signatures/contracts used

- **`GitRunner`** is the single entry point for every git
  subprocess the daemon launches. It is built once per
  process via `GitRunner::new(&Config)` and cloned cheaply
  (the inner state is `Arc<GitRunnerInner>`). Every
  invocation flows through `GitRunner::run` (or its
  `run_in(_cfg, op, args, cwd)` overload for cwd-pin
  callers); `run_args` is the stringly-typed convenience.
- **`GitRunner::with_allowlist(cfg, extras)`** lets callers
  pass an additional env-var allowlist on top of the
  baseline `PATH`, `HOME`, `USER`, `LANG`, `LC_ALL`,
  `TERM`, `TMPDIR` documented in CONTRACTS.md.
- **`GitRunner::cancel()`** flips an `AtomicBool` that
  every in-flight `run` consults on its 20 ms poll loop;
  timeout or cancel triggers `nix::sys::signal::killpg`
  against the child's process group so any grandchild
  (`ssh-askpass`, `git-credential-…`) is reaped alongside
  the parent.
- **`GitOutput { stdout, stderr, status, timed_out, cancelled }`**
  is the always-populated return — the runner never
  panics, and `timed_out` / `cancelled` are explicit
  fields so callers don't have to guess from exit codes.
- **`GIT_OUTPUT_BYTE_CAP`** (32 KiB) is the bounded stdout /
  stderr cap. Captured text longer than the cap is
  truncated to the last valid UTF-8 boundary and
  appended with `"\n...<truncated N bytes>"`.
- **`RepositoryInfo { path, base_branch, remote_url }`** is
  the canonical return of `find_main_clone`.
- **`find_main_clone(cfg, runner, key) ->
  CaduceusResult<RepositoryInfo>`** validates that the
  on-disk clone at `<workdir_base>/<owner>/<repo>` is a
  clean git working tree whose origin host matches the
  daemon's `api_base` host (public `github.com` or the
  enterprise host verbatim) and whose normalized owner /
  repo matches the issue slug.
- **`parse_origin(remote_url)`** normalises SSH
  (`git@host:owner/repo[.git]`) and HTTPS / `git://` URLs
  into `(owner, repo)`. SSH host aliases
  (`git@github.com-attacker:…`) parse to a host string
  that `validate_origin_host` then refuses; the alias
  rejection is the v0.1 mitigation called out in
  CONTRACTS.md.
- **`clone_path(cfg, key) -> PathBuf`** is the path
  arithmetic helper — components come straight from the
  (validated) `IssueKey`, no path traversal is possible.

## Behaviour

- **Process-group isolation.** Each git subprocess is
  spawned with `process_group(0)` (the safe-API equivalent
  of `setsid()` exposed by `tokio::process::Command`). The
  child is therefore its own session leader, and the
  runner broadcasts SIGKILL to `-pid` on timeout or
  cancel. The crate-wide `#![forbid(unsafe_code)]` is
  honoured because no raw `unsafe` block appears in the
  module — the unsafe syscalls are funneled through the
  `nix` crate's safe wrappers.
- **Prompt suppression.** Every child gets
  `GIT_TERMINAL_PROMPT=0` so a missing SSH agent or
  credential helper surfaces as a loud error instead of
  hanging the daemon.
- **Credential scrubbing.** Before the inherited env is
  applied, the runner removes `GITHUB_TOKEN`,
  `CADUCEUS_GITHUB_TOKEN`, and `GH_TOKEN` from the
  parent's environment. The denial list is also applied
  as a redaction pass on captured stderr so a token that
  escapes via a debug log line never reaches a `tracing`
  record or a test-failure message.
- **Inherited env allowlist.** The runner carries an
  explicit allowlist rather than inheriting blindly from
  the daemon's process env. The default is the
  contract-documented baseline; tests use
  `with_allowlist` to thread `PATH` (or other names)
  through.
- **Bounded stdout / stderr with redaction.**
  `redact_and_cap` is the single sink: it scrubs via
  `error::scrub` (the same helper that protects
  `CaduceusError` fields) and then byte-caps via
  `cap_text`. UTF-8 boundaries are respected during
  truncation so multi-byte characters never land
  half-clipped.
- **Repository discovery.** `find_main_clone` runs the
  documented checks in order: path exists → path is a
  directory → `git rev-parse --git-dir` succeeds → `git
  status --porcelain` is empty → `git config --get
  remote.origin.url` returns a non-empty URL → `parse_origin`
  yields a slug matching the issue key → host matches
  `api_base` → `git symbolic-ref refs/remotes/origin/HEAD`
  resolves a base branch (with the documented
  local-HEAD fallback + tracing warning when origin HEAD
  is missing).
- **Host validation.** Public github.com is detected by
  api_base being `https://api.github.com` (the
  documented default) or the parsed host being
  `api.github.com` / `github.com`; in either case the
  origin host must equal `github.com`. Enterprise API
  bases require the origin host to equal the api_base
  host verbatim. SSH aliases (any host segment
  containing a `-` that doesn't end with `github.com`)
  are rejected.

None of Phase 4's durable state moves in this task — the

which is contractually the daemon's sole mutator of the
on-disk clones.

## Forbidden-side-effect checks

Per `CONTRACTS.md` invariants and the Phase 04 gate:

- **#6 (worker-session kill)** — the runner uses
  `process_group(0)` and `nix::sys::signal::killpg` to
  broadcast SIGKILL to the entire process group. The
  test `git_runner_kills_process_group_on_timeout`
  exercises a long-running grandchild and observes
  `timed_out = true` within 10 s of the 1 s ceiling.
- **#7 (Rust owns heartbeats / process lifecycle)** —
  the runner is Rust-only; no Python subprocess.
- **#8 (no credential leakage)** — the runner's
  `DENIED_INHERITED_VARS` list removes
  `GITHUB_TOKEN` / `CADUCEUS_GITHUB_TOKEN` / `GH_TOKEN`
  from the parent env before launching the child. The
  test `git_runner_does_not_inherit_github_token`
  sets the three values in the parent and asserts none
  appear in the child's stdout. The runner's
  `redact_and_cap` then scrubs any token-shaped
  substring from captured stderr; the
  `git_runner_redacts_token_shaped_substrings_from_stderr`
  test confirms `ghp_should_never_leak` is replaced with
  `<redacted>`.
- **#4 (GitHub base URL credential denial)** — the
  test set deliberately includes
  `validate_origin_host_rejects_ssh_alias_like_github_com_attacker`
  so the SSH-host-alias rejection in v0.1 (which the
  contract calls out as the only safe posture) is
  verified at compile time.
- **Prompt-suppression requirement** — the test
  `git_runner_sets_git_terminal_prompt_zero_on_child`
  asserts via a `git -c alias.X='!...' X` probe that
  the child sees `GIT_TERMINAL_PROMPT=0`.
- **Stderr truncation requirement** — the test
  `git_runner_truncates_stderr_above_cap` writes 40 KiB
  to stderr via a `!python3 -c` alias and asserts the
  runner's output stays under
  `GIT_OUTPUT_BYTE_CAP + 64` bytes and contains the
  truncation marker.

## Residual risks

- The `git -c alias.X='!body' X` test seam relies on the
  host system having a `/bin/sh` capable of executing
  the alias body. CI environments without a POSIX shell
  would see the runner tests fail in the timeout /
  cancellation / redaction / truncation groups; the
  inline tests in `src/worktree.rs` (which don't shell
  out) would still pass. Phase 5's worker supervisor
  inherits the same precondition.
- `find_main_clone` exits early with a `Worktree {
  stderr: "no remote.origin.url configured at …" }`
  error when the working tree has no remote. Operators
  who routinely use a detached upstream without an
  `origin` remote will hit this and need to either set
  the remote or extend the discovery path in a future
  phase.
- The `0.04s` and `0.21s` per-test latencies (visible in
  the carrier-suite `test result` lines) are dominated
  by spawning `git` via the runner. Phase 7's tick
  budget easily absorbs the cost.
