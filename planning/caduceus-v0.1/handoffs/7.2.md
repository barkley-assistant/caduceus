## Public signatures/contracts used

- **`StateMeta`** is the canonical envelope. Fields: `version`,
  `last_tick_started`, `last_tick_finished`, `last_outcome`,
  `last_http_status`, `next_allowed_poll_at`, `last_reap_at`,
  `last_reaped_count`, `rate_limit`, `last_error`,
  `recent_diagnostics`. The struct uses `#[serde(deny_unknown_fields)]`
  for schema stability.
- **`TickOutcome`** is `{ Processed, Idle, Concurrent, Cadence,
  RateLimited, Cancelled, Failed }` with `snake_case` serde renaming.
- **`RateLimitObservation`** includes `limit`, `remaining`,
  `reset_at`, and `observed_at`. `is_newer_than` compares by
  `reset_at`.
- **`DaemonDiagnostic`** carries `timestamp`, `code`,
  `issue_key: Option<IssueKey>`, and a bounded human message.
- **`MetaStore`** — `open`, `update`, `snapshot`, `is_corrupt`,
  `clear_corrupt_marker`, and the test-seam accessors
  `meta_path`, `corrupt_marker_path`, `state_dir`.
- **`RateLimitObserver`** — `new(&MetaStore)`, `observe(observation)`.
- **`load`**, **`save`** are the file-level helpers used by tests
  and by the future daemon tick. Production code that needs
  serialised writes uses `MetaStore::update`.
- **`append_diagnostic(meta, code, issue_key, message)`** is the
  coalescing helper. Messages are truncated to 256 bytes at a
  UTF-8 char boundary.
- **Constants** — `META_VERSION = 1`, `MAX_DIAGNOSTICS = 20`,
  `DIAGNOSTIC_COALESCE_WINDOW = 1 hour`, `META_FILENAME =
  "state_meta.json"`, `CORRUPT_MARKER_FILENAME = "state_meta.corrupt"`.

## Behaviour

- **Concurrency.** `MetaStore::update` acquires the internal
  mutex before calling the closure, then serialises the result
  through the atomic-rename writer. Concurrent updates to
  different fields never lose a field; concurrent
  `RateLimitObserver::observe` calls converge on the
  observation with the newest `reset_at`.
- **Atomic write.** `save_atomic` writes to `<path>.tmp`, fsyncs,
  and renames into place. The `.tmp` is cleaned up by the
  `rename` failure path (no partial file leaks).
- **Corrupt handling.** A file that exists but cannot be parsed,
  or whose `version` differs from `META_VERSION`, is preserved;
  a timestamped copy `state_meta.json.corrupt-<ts>` is created;
  the `<state_dir>/state_meta.corrupt` marker is written with a
  brief reason; the original file is left in place. The error
  surfaces as `CaduceusError::StateCorrupt` with the path and a
  message that names the parse failure or the version mismatch.
  Later ticks see the marker and refuse to call GitHub.
- **Diagnostic coalescing.** A new entry that matches an existing
  `(code, issue_key)` within one hour refreshes the existing
  entry's `timestamp` and `message`; it does not grow the file.
  Different codes, different issue keys, or entries older than
  one hour produce a new entry. The list is capped at the 20
  newest entries.
- **Logging.** A future tightening of the logging policy can be
  applied at the single helper used by the structured logger;
  this commit does not change the logger.

## Forbidden-side-effect checks

Per CONTRACTS.md and the Phase 01 gate:

- `MetaStore::update` is the only mutating entry point that
  persists state. Concurrent calls are serialised by the mutex;
  no field is ever silently overwritten.
- The atomic-rename strategy ensures the active file is either
  the old version or the new version, never a partial file.
- Corrupt files are *preserved*, not deleted. The contract's
  "preserve corruption for diagnosis" rule is enforced.
- A stale observation cannot overwrite a newer one — the
  observer refuses by comparing `reset_at`.

## Residual risks

- The corrupt marker is honoured by future tick code that has
  not been written yet. Phase 7's `run` task will refuse GitHub
  calls while the marker exists; the helper
  `MetaStore::is_corrupt` is the entry point.
- `RateLimitObserver::observe` is bounded by the per-call mutex
  lock. A future optimisation could use a `RwLock` to allow
  concurrent reads; today's contract does not require it.
