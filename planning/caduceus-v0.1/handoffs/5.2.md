## Public signatures/contracts used

- **`SanitizedEnvInputs { issue, issue_title, issue_body, labels, worktree_path, run_id, branch_name, allowlist, context_json }`**
  is the canonical inputs struct. The contract pin is "all
  `CADUCEUS_*` variables set in *inputs* are layered on top of
  the parent, so a worker-visible variable never inherits from
  the inherited env".
- **`sanitized_env(parent: &BTreeMap<OsString, OsString>, inputs: &SanitizedEnvInputs) -> CaduceusResult<BTreeMap<OsString, OsString>>`**
  is the single allowlist-and-denylist authority. The function
  is pure: parent and inputs are passed in so tests can drive
  every branch without mutating the process env.
- **`spawn(command: &[String], cwd: &Path, inputs: &SanitizedEnvInputs) -> CaduceusResult<Command>`**
  is the canonical production spawn path. It calls
  `Command::env_clear()` and then `envs(sanitized_env(...))`, so
  the only env the child ever sees is the sanitized set.
- **`DEFAULT_ALLOWLIST_EXACT: &[&str]`** is the eight-name exact
  allowlist (`PATH`, `HOME`, `USER`, `SHELL`, `LANG`, `LC_ALL`,
  `TERM`, `TMPDIR`).
- **`DEFAULT_ALLOWLIST_PREFIXES: &[&str]`** is the four-pattern
  prefix allowlist (`OPENAI_*`, `ANTHROPIC_*`, `OPENROUTER_*`,
  `OPENCODE_*`).
- **Deny-by-default contract.** The deny list is the union of
  four exact names (`GITHUB_TOKEN`, `GH_TOKEN`,
  `CADUCEUS_GITHUB_TOKEN`, `AUTO_ISSUE_GITHUB_TOKEN`), any
  variable whose name contains both `GITHUB` and `TOKEN` as
  substrings, and any `CADUCEUS_*` variable whose name contains
  `SECRET` or `TOKEN` (the daemon-internal-secret clause). The
  ordering in `sanitized_env` is *deny first, then allow*: a
  credential name the operator adds to the allowlist is still
  dropped.
- **Labels and paths.** `CADUCEUS_ISSUE_LABELS_JSON` is the
  `serde_json::to_string(&inputs.labels)` rendering, so commas
  and quotes survive the env boundary intact. `CADUCEUS_WORKTREE_PATH`
  is the absolute UTF-8 form of `inputs.worktree_path`; a
  relative or non-UTF-8 path returns a `CaduceusError::Config`
  so the bridge's `os.path` calls stay deterministic.
- **Sanitized-env layering.** The eight `CADUCEUS_*`
  contract variables (`CADUCEUS_ISSUE_NUMBER`,
  `CADUCEUS_ISSUE_TITLE`, `CADUCEUS_ISSUE_BODY`,
  `CADUCEUS_ISSUE_REPO`, `CADUCEUS_ISSUE_LABELS_JSON`,
  `CADUCEUS_WORKTREE_PATH`, `CADUCEUS_RUN_ID`,
  `CADUCEUS_BRANCH_NAME`, `CADUCEUS_CONTEXT_JSON`) are layered
  on top of the parent copy, so a `CADUCEUS_*` value the
  operator set in the shell is never trusted — the daemon
  owns them.

## State/schema effects

None. `sanitized_env` is read-only with respect to the
filesystem and the queue. It does not call git, does not open
a socket, does not spawn a subprocess. The production `spawn`
helper constructs a `std::process::Command` for the caller to
exec; the supervisor (`worker_supervisor`) owns the
timeout/process-group/parent-death plumbing, so this task
remains a pure environment-construction concern.

## Forbidden-side-effect checks

Per `CONTRACTS.md` invariants and the Phase 05 gate:

- **#5 (daemon owns branch name)** — `CADUCEUS_BRANCH_NAME` is taken
  verbatim from `inputs.branch_name`. The function does not let the
  worker select a ref; the test asserts the value matches what the
  caller passed in.
- **#6 (worker session + supervisor)** — `spawn` is the spawner,
  the supervisor owns the rest. The task packet explicitly narrows
  scope to "spawn must call `env_clear()` before `envs`", and the
  function does exactly that.
- **#7 (Rust owns heartbeats / process lifecycle)** — unchanged;
  `sanitized_env` is pure, `spawn` only constructs a
  `std::process::Command`.
- **#8 (no credential leakage)** — verified end-to-end. The deny
  list covers all four exact names plus any `*GITHUB*TOKEN*` plus any
  `CADUCEUS_*SECRET*` / `*TOKEN*`. The real-child tests plant
  `GITHUB_TOKEN`, `AWS_SECRET_ACCESS_KEY`, and `UNRELATED_INJECTED`
  in `std::env` and assert none of them reach the child. A
  `GITHUB_TOKEN` entry the operator adds to the operator allowlist
  is still denied (the deny check runs first).
- **No git, no network, no queue mutation.** `sanitized_env` is a
  pure function over an in-memory `BTreeMap`; `spawn` only constructs
  a `Command`. No git subprocess, no HTTP request, no queue/state
  write.
- **No env mutation in tests.** Two tests use `std::env::set_var`
  / `remove_var` to plant credentials in the inherited env; the
  setup and teardown are paired, and the values are removed before
  the test returns. The runner is otherwise hermetic.

## Residual risks

- **Operator-prefix denial is conservative.** The contract pins the
  denial of `*GITHUB*TOKEN*`; the implementation also denies any
  `CADUCEUS_*SECRET*` / `*TOKEN*` variable. Operators that want to
  pass a non-secret variable named, say, `MY_TOKEN` through the
  worker must do so via an explicit allowlist entry. The contract
  pin is "no GitHub credential reaches the worker", and the broader
  denial is a defence-in-depth choice; a future revision could
  relax the `CADUCEUS_*SECRET*` rule if a real non-secret use case
  appears.
- **`spawn` reads the *current* process env.** The production
  `spawn` helper calls `std::env::vars_os()` once, immediately
  before `envs`. If the daemon's own environment changes between
  the call site and the supervisor's `exec`, the new spawn will
  pick up the latest values. The denial list is the canonical
  authority either way; this is just a note that the parent env
  capture is not snapshotted earlier in the daemon lifecycle.
- **Real-child tests rely on `python3` being on PATH.** Both
  end-to-end tests exec `python3 <tempdir>/dump_env.py`. The CI
  environment used by Caduceus already runs Python (the bridge
  is Python), so this is a safe assumption. Operators running
  the test suite on a stripped host without `python3` will see
  the two spawn tests fail with a `No such file or directory`
  error; this is acceptable because the rest of the daemon
  would not function without Python either.
- **`sanitized_env` returns the BTreeMap directly.** The contract
  text says "implement `sanitized_env(args) -> CaduceusResult<BTreeMap<OsString, OsString>>`".
  The implementation matches the contract verbatim. A future
  revision that wants a scrubbing `Debug` impl can wrap the
  return type without changing the call sites that already use
  the BTreeMap directly.
